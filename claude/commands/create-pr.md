# Create Pull Request

Create a pull request following our Linear workflow conventions.

## Instructions

1. Read `.cursor/rules/linear-workflow.mdc` for PR format and conventions
2. Check the current branch name for the Linear ticket number (format: `name/abc-XXXX-description`)
3. Push the branch to origin if not already pushed
4. Create a PR using `gh pr create` following the conventions in the workflow doc
5. Assign reviewers as specified in the workflow doc

## Reference

All PR conventions (title format, body format, reviewer responsibilities, rollback requirements) are defined in:

**`.cursor/rules/linear-workflow.mdc`**

Follow that document as the single source of truth.
